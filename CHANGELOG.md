# Changelog

本專案的所有顯著變更將記錄在此檔案中。
格式參考自 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.1.0/)。

## [4.0.0] - 2026-04-21

### Added

- **排行榜系統 (Leaderboard)**：新增 `/leaderboard` 指令，可查看前 10 名的報數最高紀錄。
- **個人紀錄追蹤**：現在會自動儲存每位使用者的歷史最高報數紀錄。
- **指令選單支援**：在 Telegram 對話框中新增了指令選單，方便使用者快速存取 `/leaderboard`。

### Changed

- **資料庫架構升級**：新增了 `user_stats` 資料表，用於持久化儲存使用者紀錄與排行榜資訊。
- **使用者名稱優化**：在排行榜與通知中改為使用使用者的全名 (`full_name`)，提升辨識度。

## [3.0.1] - 2026

### Added

- **支援小數無條件捨去 (Floor Rounding)**：現在報數不再強制要求算式結果為整數。所有結果都會經過 `math.floor` 處理。
    - 例如：輸入 `√2` 會被解析為 `1`，輸入 `3.999` 會被解析為 `3`。
    - 這增加了更多樣化的報數方式（如使用非完全平方數的根號）。

### Changed

- **錯誤訊息增強**：報數錯誤時，機器人現在會同時顯示「計算結果」與「捨去後的結果」，方便使用者除錯。
- **邏輯優化**：移除了舊版的整數判定邏輯，簡化了核心判斷流程。

## [3.0.0] - 2026

### Added

- **SymPy 強力解析引擎**：改用 `sympy` 函式庫取代原本的 `simpleeval`，提供專業的數學表達式解析。
- **隱含乘法支援**：現在支援如 `2(3+1)` 這種省略乘號的寫法（會解析為 8）。
- **浮點誤差容放**：增加了對微小浮點誤差的容忍度（1e-10），防止如 `3.0000000000001` 被判定為非整數的尷尬情況。

### Changed

- **安全性提升**：透過 SymPy 的 `parse_expr` 進行更嚴格的算式沙盒化解析。
- **錯誤提示優化**：改進了浮點數顯示格式（使用 `:g`），自動去掉無意義的小數點與後方的零。
- **依賴庫更新**：新增 `sympy` 作為核心數學引擎。

## [2.1.1] - 2026

### Added

- **進階數學運算支援**：新增了對平方 (`^`) 與開根號 (`√`) 的支援。
    - 使用者可以輸入如 `2^3` (8) 或 `√16` (4) 進行報數。
- **即時視覺回饋**：機器人會留下 `👍` 或 `👎` 反應及執行懲罰。

### Changed

- **正則表達式升級**：更新了解析邏輯，以支援包含 `^` 與 `√` 符號的複雜算式。
- **解析邏輯優化**：內部將 `^` 轉換為 Python 的 `**` 運算子，並將 `√` 對應至 `math.sqrt` 函式。

## [2.1.0] - 2026

### Added

- **嵌入式報數解析 (Embedded Parsing)**：現在支援從訊息中自動提取第一個算式或數字。
    - 例如：發送「目前是 5+5 喔」會自動解析出 `10`。
    - 提高了遊戲的趣味性與互動自由度，不再強制要求訊息只能包含數字。
- **正則表達式過濾**：引入 `re` 模組，精確抓取訊息中的數值區塊，並過濾掉無關的文字。

### Changed

- **解析邏輯優化**：增加了算式長度限制與例外處理，防止非法字串導致程式崩潰。

## [2.0.0] - 2026

### Added

- **異步資料庫操作**：引入 `aiosqlite` 取代原本的阻塞式 `sqlite3`，提升在高併發環境下的反應速度。
- **記憶體快取 (Caching)**：在啟動時將群組狀態載入記憶體，大幅減少磁碟 I/O 負擔。
- **嚴格整數判定**：現在會檢查數學運算結果是否為整數，若計算結果為浮點數（如 `1.5`）將判定為報數錯誤並重置。

### Changed

- **程式架構優化**：重構資料庫初始化邏輯至 `post_init` 非同步函式中，確保機器人啟動流程更穩健。
- **錯誤提示優化**：改進了報數錯誤時的提示訊息，明確區分數字錯誤與類型錯誤。

## [1.0.0] - 2026

### Added

- **初始版本發佈**：一個簡單的 Telegram 報數機器人。
- **數學運算支援**：使用者可以使用簡單算式（如 `1+1`）進行報數，增加趣味性。
- **懲罰機制**：報數錯誤或順序錯誤的使用者將被暫時禁言。
- **報數重置**：發生錯誤時，目前的報數進度將歸零。
- **連續報數限制**：防止同一位使用者連續報數，確保多人參與。
- **資料庫持久化**：使用 SQLite 儲存各群組與 Topic 的報數狀態，確保重啟不掉進度。
- **白名單與 Topic 支援**：可限制機器人僅在特定的群組或討論主題（Topic）中運作。
